# Test Generate — Генерация визуальных проверок

> **Роль:** Senior QA Engineer  
> **Назначение:** Шаблон для генерации тест-кейсов на визуальную компоновку и верстку  
> **Применение:** Используется LLM для создания проверок соответствия UI макету

---

## Переменные

| Переменная | Тип | Описание | Пример |
|------------|-----|----------|--------|
| `$SCREEN_ID` | string | ID экрана из ТЗ | SCR-01, SCR-02 |
| `$SCREEN_NAME` | string | Название экрана | Экран авторизации |
| `$ELEMENTS` | array | Список элементов экрана | Логотип, кнопка, поле ввода |
| `$LAYOUT_SPEC` | object | Спецификация расположения | Отступы, выравнивание |
| `$BREAKPOINTS` | array | Точки адаптации | 320px, 768px, 1024px |
| `$COLOR_SCHEME` | object | Цветовая схема экрана | Primary: #FF6B35 |

---

## Правила использования

### Общие правила

1. **Проверять только визуальную компоновку** — позиционирование, отступы, выравнивание
2. **Не проверять функциональность** — она в отдельных тест-кейсах
3. **Использовать CSS-селекторы** для идентификации элементов
4. **Учитывать адаптивность** — проверять на указанных breakpoints
5. **Проверять состояния** — default, hover, focus, disabled, loading

### Приоритеты проверок

| Приоритет | Что проверять | Метод |
|-----------|---------------|-------|
| high | Корректность отображения основных элементов | Screenshot comparison |
| medium | Отступы и выравнивание относительно макета | Visual inspection |
| low | Адаптивность на разных экранах | Responsive testing |
| low | Состояния элементов (hover, focus) | Interaction testing |

## Задачи

### Задача 1: Генерация проверок позиционирования

**Вход:** SPEC.md экрана  
**Выход:** Список проверок позиционирования элементов

**Проверки:**

- [ ] Элемент X находится на позиции Y px от родителя
- [ ] Элемент выровнен по центру/лево/право
- [ ] Вертикальное выравнивание соответствует макету
- [ ] Соблюдены отступы (margin/padding) от соседних элементов

### Задача 2: Генерация проверок стилей

**Вход:** COLOR_SCHEME, typography spec  
**Выход:** Список проверок стилей

**Проверки:**

- [ ] Цвет фона соответствует схеме
- [ ] Цвет текста соответствует схеме
- [ ] Шрифт и размер соответствуют спецификации
- [ ] Интерактивные элементы имеют правильные состояния

### Задача 3: Генерация проверок адаптивности

**Вход:** BREAKPOINTS  
**Выход:** Список проверок responsive-версий

**Проверки:**

- [ ] На breakpoint 320px: мобильная версия
- [ ] На breakpoint 768px: планшетная версия
- [ ] На breakpoint 1024px: десктопная версия
- [ ] Переход между версиями плавный

## Методы проверки

| Метод | Инструменты | Когда использовать |
|-------|-------------|-------------------|
| Screenshot comparison | Chromatic, Percy, BackstopJS | Регрессия UI |
| Visual inspection | DevTools, Browserstack | Ручная проверка |
| CSS testing | Jest, Cypress | Автоматизация стилей |
| Responsive testing | Browserstack, Sauce Labs | Кроссбраузерность |

---

## Требования к LLM

1. Анализировать входной SPEC.md экрана
2. Извлекать элементы и их спецификацию
3. Генерировать проверки по шаблону выше
4. Приоритизировать проверки (high-low)
5. Учитывать все breakpoints из спецификации
6. Использовать CSS-селекторы для идентификации
