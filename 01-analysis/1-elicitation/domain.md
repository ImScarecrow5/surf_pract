# Описание домена и ограничения

## Описание домена

**Бизнес.** «Вертикаль» — скалодром в бывшем складском ангаре, работающий третий год.
Клуб организует **групповые тренировки** по скалолазанию. Тренировка длится 1.5 часа и проходит
по одной из **зон**: болдеринг — для новичков, трассы с верёвкой — для опытных. Каждую
тренировку ведёт **инструктор** (сейчас их пятеро, в сезон нанимаются подработчики).

**Процесс.** Владелец заранее формирует **расписание на неделю вперёд**: для каждой тренировки
задаются дата и время старта, зона, назначенный инструктор и число мест. Для болдеринга берут
не более 8 человек (нужно больше следить по технике безопасности), для трасс с верёвкой — до
16 человек. Клиент видит свободные **слоты** тренировок и **записывается**, указывая, со своим
снаряжением или возьмёт **напрокат**. До тренировки клиент может **отменить** запись — при
ранней отмене место освобождается, поздняя отмена влечёт штраф. Сейчас запись клиентов
ведётся вручную (Telegram + тетрадь), что приводит к двойным записям и путанице, особенно в
часы пик.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на тренировки
(просмотр и фильтрация слотов, бронирование, выбор проката, отмена, свои бронирования,
оценка инструктора), исключив ручную путаницу и двойные брони. Управление расписанием и
работу инструкторов/владельца обеспечивает **существующая инфраструктура**; приложение
потребляет её данные через API.

**Ключевые объекты домена:**

- **Тренировка / слот** — конкретное занятие: дата/время старта, зона, инструктор, всего и свободно мест. Создаётся в существующей инфраструктуре; клиенту доступен только для чтения.
- **Зона / формат** — вариант тренировки (болдеринг / трассы с верёвкой), определяет длительность и потолок мест (справочные данные, read-only).
- **Инструктор** — ведёт тренировку; для клиента — справочные данные слота (read-only), с рейтингом.
- **Снаряжение** — инвентарь скалодрома (скальники, страховочная система, обувь); может выдаваться напрокат.
- **Бронирование** — резерв места клиентом на конкретный слот: своё/прокатное снаряжение, статус.
- **Клиент** — записывается на тренировки, отменяет запись, оценивает инструкторов.

## Пользователи и роли

> **Скоуп текущей работы — только клиентское веб-приложение и API для него.**
> Роли Инструктор и Владелец/Администратор обслуживаются **существующей инфраструктурой**
> (бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Гость скалодрома, занимается лазанием. Пользуется с телефона (мобильная версия сайта). | Увидеть и отфильтровать свободные слоты, записаться, выбрать своё/прокатное снаряжение, видеть свои бронирования, отменить запись, получать напоминания, оценить инструктора. | **Да** |

> Инструктор и Владелец/Администратор взаимодействуют с системой через **уже реализованные**
> интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные
> ими данные (слоты, зоны, инструкторы) через API.

## Ограничения

**Бизнес-ограничения**

- Болдеринг — **не более 8 человек** (для новичков, больше внимания по технике безопасности).
- Трассы с верёвкой — **до 16 человек** (для опытных).
- **Политика отмен:** бесплатная отмена не позднее чем за **2 часа** до старта; поздняя
  отмена (<2 ч) — штраф **10% от стоимости**.
- **Тайм-лимит подтверждения брони — 15 минут.** Бронирование должно быть подтверждено в
  течение 15 минут, иначе место освобождается.
- **Слоты с 0 свободных мест скрыты.**
- **Новички видят только болдеринг.**
- **Оплата офлайн** (наличные / перевод на карту) — приложение лишь фиксирует запись и
  показывает цену.
- **Период списка слотов по умолчанию — 7 дней.** По умолчанию клиенту показываются
  слоты на **ближайшие 7 дней** от текущего момента — это совпадает с горизонтом планирования
  расписания («на неделю вперёд»). Более длинный период доступен через **фильтр дат**. При
  отсутствии расписания на ближайшие дни показывается **empty state** («Пока нет доступных
  тренировок»).
- **Отмена тренировки скалодромом / по профилактике.** Если тренировку отменяет скалодром
  (организатор или из-за профилактики/проверки снаряжения), бронь на стороне клиента **не
  удаляется**, а получает статус **«Отменена скалодромом»** с указанием причины; клиенту
  приходит **push-уведомление**. **Повторная запись на отменённый слот запрещена.** Само
  действие отмены слота выполняется в существующей инфраструктуре (вне скоупа клиентского
  приложения).

**Технические / платформенные**

- **Mobile-first**: клиентское приложение — веб-сайт, адаптированный под мобильные устройства.
- Клиентское приложение взаимодействует с **существующим бэкендом через API**; слоты, зоны и
  инструкторы доступны только для чтения.
- **Граница серверной интеграции.** Детали серверной реализации — транзакционность,
  SLA, маппинг сущностей на внутренние модели бэкенда — **вне скоупа клиентского приложения**.
  Существующий бэкенд — **black-box источник истины**: гарантия «**0 двойных броней**»
  обеспечивается на его стороне (атомарная проверка свободных мест при бронировании). Со стороны
  клиента граница закрывается **контрактом API**: приложение полагается на ответы бэкенда и
  корректно обрабатывает отказ, если мест уже нет.
- **Данные существующей инфраструктуры.** Это **учебный/тестовый проект**, исторических
  данных и легаси-схемы нет. **Канонической схемой считается контракт API**: бэкенд по условию
  отдаёт ровно те поля, что описаны в контракте. Поэтому **миграция/backfill и стратегия
  дозаполнения исторических слотов вне скоупа** — отдельных требований к ним не формулируем.

**Проект**

- **Срок** — к началу следующего сезона, ~2 месяца.
- **Бюджет** ограничен → начинаем с самого нужного, «навороты» позже.

**Границы скоупа**

Скоуп — **клиентское веб-приложение и API для него**. Управление расписанием, роли
инструктора и владельца, админка — это **существующая инфраструктура**, вне скоупа.

| В скоупе клиентского приложения | Вне скоупа (существующая инфраструктура / не делаем сейчас) |
| :-- | :-- |
| Регистрация/вход клиента по телефону | Роли Инструктора и Владельца/Администратора, админ-экран |
| Просмотр и **фильтрация** списка слотов (дата, зона, наличие мест, инструктор) | Создание и редактирование расписания (слоты приходят из существующего бэкенда через API) |
| Самостоятельная запись | Оценка инструктора — **Phase 2** |
| Выбор «своё / прокатное снаряжение», контроль лимита мест и прокатного фонда | Отметка явки/неявки, отмена слота инструктором + массовое уведомление |
| Список своих бронирований | Онлайн-оплата; ручная отметка «оплачено» |
| Отмена записи с правилом 2 часов и штрафом 10% | Программа лояльности для постоянных клиентов |
| Цена и фиксация записи (оплата офлайн) | Публичный показ рейтинга инструкторов клиентам — **Phase 2** |
| Напоминания о записи — за 1 час до тренировки | |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Тренировка / слот** | Запланированное групповое занятие: дата/время старта, зона, инструктор, всего и свободно мест. |
| **Зона / формат** | Вариант тренировки: болдеринг (для новичков, потолок 8) или трассы с верёвкой (для опытных, до 16). |
| **Болдеринг** | Скалолазание без верёвки на невысоких стенах; для новичков, максимум 8 человек. |
| **Трассы с верёвкой** | Скалолазание с верхней страховкой; для опытных, до 16 человек. |
| **Снаряжение** | Инвентарь скалодрома: скальники, страховочная система, обувь. Может выдаваться напрокат. |
| **Своё снаряжение** | Клиент приезжает со своим снаряжением; занимает место в группе, но не прокатный фонд. |
| **Прокат** | Выдача клубного снаряжения клиенту, у которого нет своего. |
| **Бронирование** | Резерв места клиентом на конкретный слот: своё/прокатное снаряжение, статус. |
| **Отмена** | Снятие записи. Клиентом: ранняя (≥2 ч до старта) — бесплатно, освобождает место; поздняя (<2 ч) — штраф 10%. Скалодромом: бронь переходит в статус «Отменена скалодромом» с причиной, клиент получает push, повторная запись на этот слот запрещена. |
| **Отмена скалодромом** | Отмена тренировки организатором или из-за профилактики/проверки снаряжения (выполняется в существующей инфраструктуре). На стороне клиента бронь сохраняется со статусом «Отменена скалодромом» + причина; отправляется push-уведомление; повторная запись на отменённый слот недоступна. |
| **Администратор / владелец** | Оля — управляет расписанием и инструкторами через существующую инфраструктуру (вне скоупа клиентского приложения). |
| **MVP** | Минимально жизнеспособный продукт: набор функций к старту сезона. |
