## Обзор

Главный экран приложения с двумя вкладками: "Расписание" для просмотра доступных слотов и "Мои бронирования" для просмотра бронирований клиента. Пользователь может переключаться между вкладками и видеть актуальную информацию.

### User Story

> Как **зарегистрированный клиент**, я хочу **видеть доступные слоты для записи и свои бронирования на одном экране**, чтобы **быстро забронировать тренировку и управлять бронированиями**.

### Бизнес-ценность

- Быстрый доступ к доступным слотам для записи
- Удобный просмотр и управление бронированиями
- Снижение количества пропущенных тренировок

### /slots

**Тип:** REST  
**Метод:** GET  
**Спецификация:** [openapi.yaml](../api/openapi.yaml) → `getSlots`

**Триггер:** Инициализация / Pull-to-refresh / Смена фильтров

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `zone` | string | Нет | Состояние / Ввод | Фильтр по зоне (boulder/rope/all) |
| `dateFrom` | string | Нет | Состояние | Начальная дата диапазона |
| `dateTo` | string | Нет | Состояние | Конечная дата диапазона |
| `instructorId` | string | Нет | Состояние | Фильтр по инструктору |
| `onlyAvailable` | boolean | Нет | — | Только со свободными местами |
| `page` | integer | Нет | — | Номер страницы |
| `limit` | integer | Нет | — | Количество элементов |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Скелетон карточек (6-8 шт.) |
| Успех | `data` не пуст | Отобразить слоты |
| Успех | `data` пуст | Empty state "Нет доступных слотов" |
| HTTP 401 | — | Redirect на SCR-01 |
| Сеть | Нет соединения | Error state с кнопкой "Обновить" |

## Элементы экрана

> **Примечания:**
> - **Колонка "Валидация":** Для полей ввода указать правило и текст ошибки. Для остальных элементов — "—".
> - **Логика:** Описывается после таблицы каждого блока в виде текстового блока "**Логика:**". Если элемент использует переиспользуемую логику из раздела [Логики](Логики/_INDEX.md), укажите ссылку на неё.
> - **Условия доступности:** Для кнопок и интерактивных элементов указать условия активности/видимости после таблицы.

### 1. Header

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Заголовок | "Расписание" или "Мои бронирования" | `selectedTab` | — | — |
| Кнопка фильтров | Иконка фильтра | — | — | Открыть BS-01 |

**Логика:**
- Заголовок: Динамический в зависимости от выбранной вкладки

**Условия доступности:**
- Кнопка фильтров видна только на вкладке "Расписание"

### 2. Bottom Navigation

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Вкладка "Расписание" | Иконка + текст | — | — | Переключение на вкладку слотов |
| Вкладка "Мои бронирования" | Иконка + текст | — | — | Переключение на вкладку бронирований |
| Индикатор активной вкладки | Подсветка/подчёркивание | `selectedTab` | — | — |

**Логика:**
- При тапе на вкладку: Загрузка соответствующих данных, смена индикатора

### 3. Вкладка "Расписание"

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Переключатель зон | Segmented Control | `zone` из `/zones` | — | Фильтрация |
| Заголовок секции (дата) | Текст | Группировка из `/slots` | — | — |
| Карточка слота | Card | `slot` из [/slots](#slots) | — | Открыть [SCR-03](SCR-03_Детали_слота.md) |
| Индикатор мест | Бейдж | `freePlaces` из слота | — | Показывает < 3 мест |
| Пустое состояние | Заглушка | — | — | — |

**Логика:**
- Карточка слота: Тап передаёт `slotId` и открывает SCR-03
- Для `client.level = beginner`: показывать только слоты болдеринга (boulder)
- Индикатор мест: красный текст при `freePlaces < 3`

**Условия доступности:**
- Карточка недоступна при `freePlaces = 0`
- Карточки с 0 мест скрыты или показаны как недоступные

### 4. Вкладка "Мои бронирования"

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Переключатель вида | Segmented Control | `status` | — | Фильтрация |
| Карточка бронирования | Card | `booking` из [/bookings](#bookings) | — | Открыть [SCR-06](SCR-06_Детали_бронирования.md) |
| Статус бронирования | Бейдж | `status` из бронирования | — | Цвет: success/error/warning |
| Пустое состояние | Заглушка | — | — | — |

**Логика:**
- Карточка бронирования: Тап передаёт `bookingId` и открывает SCR-06
- Статус отображается цветом: confirmed=#4CAF50, cancelled_by_*=#F44336, cancelled_by_gym=#9E9E9E

**Условия доступности:**
- Кнопка "Отменить" видна только для статуса `confirmed`

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** авторизованный пользователь, **Когда** открытие экрана, **Тогда** загружаются слоты и бронирования | P0 |
| AC-002 | **Дано** слоты найдены, **Когда** загрузка, **Тогда** отображаются карточки слотов | P0 |
| AC-003 | **Дано** тап на карточку слота, **Тогда** открывается SCR-03 с деталями | P0 |
| AC-004 | **Дано** тап на вкладку "Мои бронирования", **Тогда** загружаются бронирования | P0 |
| AC-005 | **Дано** pull-to-refresh, **Тогда** данные обновляются | P1 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** ошибка сети, **Когда** открытие экрана, **Тогда** отображается error state с кнопкой "Обновить" | P0 |
| AC-N02 | **Дано** нет доступных слотов, **Тогда** отображается empty state | P1 |
| AC-N03 | **Дано** нет бронирований, **Тогда** отображается empty state "У вас пока нет бронирований" | P1 |
| AC-N04 | **Дано** уровень "новичок", **Тогда** показываются только слоты болдеринга | P0 |

### Граничные условия (Edge Cases)

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** слоты загружены, **Когда** переключение фильтра зоны, **Тогда** список обновляется | P1 |
| AC-E02 | **Дано** много слотов, **Тогда** реализована пагинация | P2 |
| AC-E03 | **Дано** потеря сети во время запроса, **Когда** восстановление, **Тогда** возможность retry | P1 |
