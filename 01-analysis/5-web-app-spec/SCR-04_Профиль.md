## Обзор

Экран профиля клиента с тремя вкладками (табы): Профиль, История, Настройки уведомлений. Клиент может редактировать свои данные, просматривать историю бронирований и управлять настройками уведомлений.

### User Story

> Как **клиент**, я хочу **редактировать профиль, просматривать историю бронирований и управлять настройками уведомлений**, чтобы **обновить свои данные и получать уведомления в удобное время**.

### Бизнес-ценность

- Улучшение качества данных клиентов
- Снижение количества пропущенных тренировок через уведомления
- Повышение вовлечённости клиентов

## Используемые запросы

> Все API-запросы экрана с полным описанием параметров и обработки ответов.

### GET /profile

**Тип:** REST  
**Метод:** GET  
**Спецификация:** [openapi.yaml](../api/openapi.yaml) → `getProfile`

**Триггер:** Инициализация / Pull-to-refresh

**Параметры:** —

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Скелетон на весь экран |
| Успех | 200 OK | Отобразить данные профиля |

| Сеть | Нет соединения | Error state с кнопкой "Обновить" |

### PATCH /profile

**Тип:** REST  
**Метод:** PATCH  
**Спецификация:** [openapi.yaml](../api/openapi.yaml) → `updateProfile`

**Триггер:** Тап на кнопку "Сохранить" при редактировании

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `name` | body (string) | Нет | Ввод пользователя | Имя клиента |
| `level` | body (string) | Нет | Выбор | beginner/intermediate/advanced |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Лоадер на кнопке |
| Успех | 200 OK | Обновить данные, показать success |
| HTTP 400 | Ошибка валидации | Снек "Ошибка валидации" |
| HTTP 401 | — | Redirect на SCR-01 |
| Сеть | Нет соединения | Снек "Нет соединения" |

---

**Тип:** REST  
**Метод:** GET  
**Спецификация:** [openapi.yaml](../api/openapi.yaml) → `getBookings`

**Обработка ответа:**

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** авторизованный пользователь, **Когда** открытие экрана, **Тогда** загружаются данные профиля | P0 |
| AC-002 | **Дано** профиль загружен, **Тогда** отображаются имя, телефон, уровень | P0 |
| AC-003 | **Дано** тап "Ред.", **Тогда** показываются поля для редактирования | P0 |
| AC-004 | **Дано** редактирование и сохранение, **Тогда** данные обновляются | P0 |
| AC-005 | **Дано** вкладка "История", **Тогда** показывается статистика и прошедшие бронирования | P0 |
| AC-006 | **Дано** переключатели уведомлений, **Тогда** настройки сохраняются | P1 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** ошибка сети, **Когда** открытие экрана, **Тогда** error state с кнопкой "Повторить" | P0 |
| AC-N02 | **Дано** нет истории бронирований, **Тогда** empty state | P1 |
| AC-N03 | **Дано** 401 при запросе профиля, **Тогда** редирект на авторизацию | P0 |

### Граничные условия (Edge Cases)

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** длинное имя, **Тогда** обрезка с многоточием | P2 |
| AC-E02 | **Дано** много бронирований в истории, **Тогда** пагинация | P2 |
