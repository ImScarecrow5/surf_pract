## Обзор

Список бронирований клиента: пользователь может просматривать как предстоящие (upcoming), так и прошедшие (past) бронирования с фильтрацией по статусу.

### User Story

> Как **клиент**, я хочу **просмотреть список бронирований**, чтобы **не пропустить тренировки**.

### Бизнес-ценность

- Удобный просмотр запланированных тренировок
- Снижение количества пропущенных тренировок
- Прозрачность бронирований для клиента

## Используемые запросы

> Все API-запросы экрана с полным описанием параметров и обработки ответов.

### GET /bookings

**Тип:** REST  
**Метод:** GET  
**Спецификация:** [openapi.yaml](../api/openapi.yaml) → `getBookings`

**Триггер:** Инициализация / Pull-to-refresh / Смена фильтра

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `status` | query (string) | Нет | Состояние | Фильтр по статусу |
| `upcoming` | query (boolean) | Нет | Состояние | Только предстоящие |
| `past` | query (boolean) | Нет | Состояние | Только прошедшие |
| `page` | query (integer) | Нет | — | Номер страницы |
| `limit` | query (integer) | Нет | — | Количество элементов |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Скелетон |
| Успех | `data` не пуст | Отобразить бронирования |
| Успех | `data` пуст | Empty state "Нет бронирований" |
| HTTP 401 | — | Redirect на SCR-01 |
| Сеть | Нет соединения | Error state с кнопкой "Обновить" |

## Элементы экрана

> **Примечания:**
> - **Колонка "Валидация":** Для полей ввода указать правило и текст ошибки. Для остальных элементов — "—".
> - **Логика:** Описывается после таблицы каждого блока в виде текстового блока "**Логика:**". Если элемент использует переиспользуемую логику из раздела [Логики](Логики/_INDEX.md), укажите ссылку на неё.
> - **Условия доступности:** Для кнопок и интерактивных элементов указать условия активности/видимости после таблицы.

### 1. Header

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Кнопка "Назад" | Стрелка влево | — | — | Возврат на SCR-02 |
| Заголовок | "Мои бронирования" | — | — | — |

### 2. Фильтры

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Segmented Control | "Все" / "Предстоящие" / "Прошедшие" | `filter` | — | Запрос [/bookings](#bookings-get) |

**Логика:**
- При смене фильтра: новый запрос к API с соответствующими параметрами

### 3. Список бронирований

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Карточка бронирования | Card | `booking` из [/bookings](#bookings-get) | — | Открыть [SCR-06](SCR-06_Детали_бронирования.md) |
| Дата и время | Текст | `slot.startTime` | — | — |
| Зона | Текст | `slot.zone.name` | — | — |
| Инструктор | Текст | `slot.instructor.name` | — | — |
| Статус | Бейдж | `booking.status` | — | Цвет: success/error/warning |
| Пустое состояние | Заглушка | — | — | — |

**Логика:**
- Карточка: Тап передаёт `bookingId` и открывает SCR-06
- Статус отображается цветом:
  - `confirmed` = #4CAF50 (зелёный)
  - `cancelled_by_client_*` = #F44336 (красный)
  - `cancelled_by_gym` = #9E9E9E (серый)

**Условия доступности:**
- Карточки отменённых бронирований могут быть неактивны (только просмотр)

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** авторизованный пользователь, **Когда** открытие экрана, **Тогда** загружаются бронирования | P0 |
| AC-002 | **Дано** бронирования найдены, **Тогда** отображаются карточки | P0 |
| AC-003 | **Дано** тап на "Предстоящие", **Тогда** показываются только будущие бронирования | P0 |
| AC-004 | **Дано** тап на "Прошедшие", **Тогда** показываются только прошедшие бронирования | P0 |
| AC-005 | **Дано** тап на карточку, **Тогда** открывается SCR-06 | P0 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** ошибка сети, **Когда** открытие экрана, **Тогда** error state с кнопкой | P0 |
| AC-N02 | **Дано** нет бронирований, **Тогда** empty state | P1 |
| AC-N03 | **Дано** 401 при запросе, **Тогда** редирект на авторизацию | P0 |

### Граничные условия (Edge Cases)

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** много бронирований, **Тогда** пагинация | P2 |
| AC-E02 | **Дано** пустой статус фильтра, **Тогда** показываются все | P1 |
