## Обзор

Bottom Sheet для изменения количества гостей или добавления комментария к существующему бронированию.

### User Story

> Как **посетитель скалодрома**, я хочу **редактировать бронь**, чтобы **изменить количество гостей или добавить примечание**.

### Бизнес-ценность

- Гибкость в управлении бронированиями
- Возможность добавить комментарий для инструктора
- Снижение количества отмен и повторных бронирований

## Используемые запросы

### GET /bookings/{bookingId}

**Тип:** REST  
**Метод:** GET  
**Спецификация:** `bookings.yaml` → `getBookingById`

**Триггер:** Инициализация

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `bookingId` | string | Да | Входные данные | ID бронирования |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Скелетон |
| Успех | `data` не пуст | Заполнить форму данными |
| Успех | `data` пуст | Error state |
| HTTP 4xx/5xx | — | Error state с кнопкой "Повторить" |
| Сеть | Нет соединения | Error state с кнопкой "Повторить" |

---

### PATCH /bookings/{bookingId}

**Тип:** REST  
**Метод:** PATCH  
**Спецификация:** `bookings.yaml` → `updateBooking`

**Триггер:** Тап на кнопку «Сохранить»

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `bookingId` | string | Да | Входные данные | ID бронирования |
| `guestsCount` | int | Да | UI | Количество гостей |
| `comment` | string | Нет | UI | Комментарий |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Лоадер на кнопке, блокировка UI |
| Успех | — | Закрыть BS, обновить данные |
| HTTP 4xx | `message` содержит текст | Снек с текстом из `message` |
| HTTP 5xx | — | Снек «Произошла ошибка. Попробуйте позже» |
| Сеть | Нет соединения | Снек «Нет соединения. Проверьте подключение» |

> - **Логика:** Описывается после таблицы каждого блока в виде текстового блока "**Логика:**". Если элемент использует переиспользуемую логику из раздела [Логики](Логики/_INDEX.md), укажите ссылку на неё.
> - **Условия доступности:** Для кнопок и интерактивных элементов указать условия активности/видимости после таблицы.

### 1. Header

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Заголовок | Текст «Изменить бронь» | Статический | — | — |
| Кнопка закрытия | Иконка ✕ | — | — | Закрыть Bottom Sheet |

### 2. Основной контент

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Подзаголовок | Текст «Количество гостей» | Статический | — | — |
| Счётчик | Кнопка «-», значение, «+» | `currentGuests` из API | — | Изменение значения |
| Разделитель | Горизонтальная линия | — | — | — |
| Подзаголовок | Текст «Комментарий» | Статический | — | — |
| Поле комментария | Textarea | `comment` из API | Max 500 символов | Ввод текста |
| Плейсхолдер | Текст-подсказка | Статический | — | — |
| Итоговая цена | Текст | Расчёт: `basePrice + (guests * guestPrice)` | — | — |

**Логика:**
- Счётчик: [LOGIC-005](../Логики/LOGIC-005_Counter.md) — изменение значения с ограничениями
- Поле комментария: Валидация на максимум 500 символов
- Расчёт цены: `basePrice + (guestsCount * guestPrice)`

**Условия доступности:**
- Кнопка «-» активна, если: `currentGuests > 0`
- Кнопка «+» активна, если: `currentGuests < 2` И редактирование разрешено (≥2ч до старта)
- Поле комментария: доступно для редактирования
- Блок «Итоговая цена» виден, если: всегда

### 3. Footer

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Кнопка «Сохранить» | Primary Button | — | — | PATCH /bookings/{id} |
| Кнопка «Отменить бронь» | Text Button (#F44336) | — | — | Открыть BS-02 |

**Логика:**
- Кнопка «Сохранить»: [LOGIC-006](../Логики/LOGIC-006_FormSubmit.md) — валидация → отправка формы
- Кнопка «Отменить бронь»: Открытие BS-02 с параметрами бронирования

**Условия доступности:**
- Кнопка «Сохранить» активна, если: всегда (даже без изменений)
- Кнопка «Отменить бронь» активна, если: всегда (для upcoming бронирований)
- Кнопка «Отменить бронь» скрыта, если: бронь уже нельзя отменить

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** пользователь на экране деталей бронирования, **Когда** нажимает «Изменить», **Тогда** открывается Bottom Sheet с данными | P0 |
| AC-002 | **Дано** BS открыт, **Когда** изменяет количество гостей, **Тогда** итоговая цена пересчитывается | P0 |
| AC-003 | **Дано** BS открыт, **Когда** вводит комментарий >500 символов, **Тогда** ввод обрезается до 500 | P0 |
| AC-004 | **Дано** BS открыт, **Когда** нажимает «Сохранить», **Тогда** изменения сохраняются и BS закрывается | P0 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** ошибка сети при загрузке, **Когда** открытие BS, **Тогда** error state с кнопкой «Повторить» | P0 |
| AC-N02 | **Дано** ошибка сети при сохранении, **Когда** нажатие «Сохранить», **Тогда** снек об ошибке | P0 |
| AC-N03 | **Дано** менее 2ч до старта, **Когда** открытие BS, **Тогда** счётчик гостей заблокирован | P1 |

### Граничные условия (Edge Cases)

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** комментарий уже был, **Когда** открытие BS, **Тогда** поле заполнено текущим комментарием | P1 |
| AC-E02 | **Дано** достигнут максимум гостей, **Когда** нажатие «+», **Тогда** кнопка неактивна | P1 |
