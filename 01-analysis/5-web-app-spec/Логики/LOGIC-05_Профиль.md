## Обзор

Логика управления профилем включает просмотр данных профиля (имя, телефон, уровень), редактирование имени и уровня, а также выход из аккаунта с очисткой данных.

### User Story

> Как клиент скалодрома, я хочу просматривать и редактировать свой профиль (имя, уровень), чтобы мои данные были актуальны для подбора подходящих тренировок.

### Бизнес-ценность

- Просмотр личных данных
- Обновление информации
- Настройка уровня для персонализации
- Безопасный выход из аккаунта

## Описание логики

### Шаг 1: Загрузка профиля

При открытии экрана профиля загружаем данные через GET /profile. Отображаем:
- Имя пользователя
- Номер телефона
- Уровень (новичок/опытный/продвинутый)
- Дата регистрации

### Шаг 2: Редактирование профиля

Пользователь может изменить:
- Имя (текстовое поле, макс 100 символов)
- Уровень (выбор из: beginner, intermediate, advanced)

### Шаг 3: Сохранение изменений

PATCH /profile с полями name и/или level. При успехе — обновление локальных данных.

### Шаг 4: Выход из аккаунта

Подтверждение выхода, очистка всех токенов и данных, переход на экран входа.

## API запросы

### GET /profile

**Триггер:** Открытие экрана профиля

**Headers:**

| Поле | Описание |
|------|----------|
| authorization | Bearer токен |
| deviceuuid | ID устройства |

**Параметры:** Нет

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Показ скелетона |
| Успех (200) | Отображение данных профиля |
| Ошибка 401 | Переход на авторизацию |
| Ошибка 5xx | Снек "Ошибка загрузки" |

---

### PATCH /profile

**Триггер:** Тап "Сохранить"

**Headers:**

| Поле | Описание |
|------|----------|
| authorization | Bearer токен |
| deviceuuid | ID устройства |
| Content-Type | application/json |

**Параметры/Body:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| name | string | Имя клиента (опционально) | Из поля ввода |
| level | string | beginner/intermediate/advanced | Из выбора |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Блокировка кнопки, лоадер |
| Успех (200) | Снек "Сохранено", обновить данные |
| Ошибка 400 | Снек с текстом валидации |
| Ошибка 401 | Переход на авторизацию |

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** Пользователь открывает профиль, **Когда** Данные загружены, **Тогда** Показываются имя, телефон и уровень |
| AC-002 | **Дано** Пользователь нажимает "Редактировать", **Когда** Открывается форма, **Т_then** Поля доступны для изменения |
| AC-003 | **Дано** Пользователь меняет имя, **Когда** Нажимает "Сохранить", **Т_then** Имя обновляется и показывается подтверждение |
| AC-004 | **Дано** Пользователь меняет уровень, **Когда** Выбирает новый уровень, **Т_then** При сохранении уровень обновляется |
| AC-005 | **Дано** Пользователь нажимает "Выйти", **Когда** Подтверждает выход, **Т_then** Все данные очищаются и показывается экран входа |
| AC-006 | **Дано** Имя пустое, **Когда** Пользователь сохраняет, **Т_then** Показывается ошибка валидации |
