## Обзор

Логика управления бронированиями включает просмотр списка бронирований с фильтрацией по статусу и времени (предстоящие/прошедшие), просмотр деталей конкретного бронирования и отмену бронирования с расчётом штрафа.

### User Story

> Как клиент скалодрома, я хочу видеть свои бронирования и иметь возможность отменить их при необходимости, чтобы управлять своим расписанием тренировок.

### Бизнес-ценность

- Просмотр истории бронирований
- Фильтрация по статусу и времени
- Отмена с корректным расчётом штрафа
- Прозрачная информация об отменах

## Описание логики

### Шаг 1: Загрузка списка бронирований

При открытии экрана загружаем бронирования с возможными фильтрами:
- all / upcoming / past — по времени
- status — по статусу (confirmed, cancelled и т.д.)

### Шаг 2: Отображение списка

Бронирования отображаются в виде карточек:
- Дата и время
- Зона
- Инструктор
- Статус (подтверждено/отменено)
- Цена

### Шаг 3: Просмотр деталей

При тапе на бронирование — загрузка полных деталей:
- Вся информация о слоте
- Информация о снаряжении
- Статус и история изменений
- Сумма штрафа (если применимо)

### Шаг 4: Отмена бронирования

При отмене:
- Если до старта > 2 часов — бесплатно
- Если до старта < 2 часов — штраф 10%
- Загружается PATCH /bookings/{id}/cancel

### Шаг 5: Обновление списка

После отмены — обновление списка с учётом нового статуса

## API запросы

### GET /bookings

**Триггер:** Открытие экрана "Мои брони"

**Headers:**

| Поле | Описание |
|------|----------|
| authorization | Bearer токен |
| deviceuuid | ID устройства |

**Параметры:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| status | string | Фильтр по статусу | Из выбора (confirmed/cancelled_*/all) |
| upcoming | boolean | Только предстоящие | При выборе "Предстоящие" |
| past | boolean | Только прошедшие | При выборе "Прошедшие" |
| page | integer | Страница | 1 |
| limit | integer | Количество | 20 |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Показ скелетона |
| Успех (200) | Отображение списка |
| Ошибка 401 | Переход на авторизацию |
| Ошибка 5xx | Снек "Ошибка загрузки" |

---

### GET /bookings/{bookingId}

**Триггер:** Тап на бронирование

**Headers:**

| Поле | Описание |
|------|----------|
| authorization | Bearer токен |
| deviceuuid | ID устройства |

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| bookingId | string | UUID бронирования |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Успех (200) | Показать детали |
| Ошибка 404 | Снек "Бронирование не найдено" |

---

### PATCH /bookings/{bookingId}/cancel

**Триггер:** Тап "Отменить бронирование"

**Headers:**

| Поле | Описание |
|------|----------|
| authorization | Bearer токен |
| deviceuuid | ID устройства |

**Параметры:**
| Параметр | Тип | Описание |
|----------|-----|----------|
| bookingId | string | UUID бронирования |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Лоадер на кнопке |
| Успех (200) | Снек "Бронирование отменено", обновить список |
| Ошибка 400 | Снек с текстом (нельзя отменить/время вышло) |
| Ошибка 404 | Снек "Бронирование не найдено" |
| Ошибка 5xx | Снек "Ошибка. Попробуйте позже" |

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** Пользователь открывает "Мои брони", **Когда** Загружаются его бронирования, **Тогда** Показываются все его бронирования |
| AC-002 | **Дано** Пользователь выбирает "Предстоящие", **Когда** Применяется фильтр, **Тогда** Показываются только будущие тренировки |
| AC-003 | **Дано** Пользователь выбирает "Прошедшие", **Когда** Применяется фильтр, **Тогда** Показываются только прошедшие тренировки |
| AC-004 | **Дано** Пользователь тапает на бронирование, **Когда** Загружаются детали, **Т_then** Показывается полная информация |
| AC-005 | **Дано** Пользователь отменяет за 3 часа до тренировки, **Когда** Он подтверждает отмену, **Тогда** Отмена бесплатная |
| AC-006 | **Дано** Пользователь отменяет за 1 час до тренировки, **Когда** Он подтверждает отмену, **Т_then** Взимается штраф 10% |
| AC-007 | **Дано** Бронирование уже отменено, **Когда** Пользователь тапает "Отменить", **Т_then** Кнопка отмены не отображается |

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| Нельзя отменить | Отмена | Снек "Нельзя отменить" |
| Время вышло | Отмена | Снек "Время для отмены истекло" |
| Не найдено | Детали | Снек, возврат к списку |
| Нет сети | Любой запрос | Снек "Нет соединения" |
