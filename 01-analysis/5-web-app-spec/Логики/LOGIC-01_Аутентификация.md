## Обзор

Процесс аутентификации пользователя в мобильном приложении скалодрома "Вертикаль" осуществляется через SMS-верификацию. Пользователь вводит номер телефона, получает код подтверждения по SMS, вводит код и получает токены доступа. Токены сохраняются локально для последующих автологинов.

### User Story

> Как клиент скалодрома, я хочу войти в приложение по номеру телефона, чтобы получить быстрый и безопасный доступ к бронированиям и профилю без запоминания пароля.

### Бизнес-ценность

- Упрощённый вход без регистрации пароля
- Мгновенная аутентификация через SMS
- Автоматический вход при повторном открытии приложения
- Безопасность через одноразовые коды

## Описание логики

### Шаг 1: Проверка токенов при запуске

При открытии приложения проверяется наличие сохранённых токенов. Если токены есть — проверяется их валидность через API или через декодирование JWT. При невалидном токене пытаемся обновить его через refresh token.

### Шаг 2: Запрос кода подтверждения

Пользователь вводит номер телефона в формате +7XXXXXXXXXX. Приложение отправляет запрос `POST /auth/request-code`. При успехе показывается экран ввода кода, при ошибке — сообщение об ошибке.

### Шаг 3: Верификация кода

Пользователь вводит 4-6 значный код из SMS. Приложение отправляет `POST /auth/verify` с телефоном и кодом. При успехе сохраняются токены и пользователь переходит на главный экран.

### Шаг 4: Сохранение токенов

При успешной верификации access token сохраняется в локальный кэш, refresh token — в защищённое хранилище. Также сохраняется время истечения токена.

## API запросы

### POST /auth/request-code

**Триггер:** Тап на кнопку "Получить код"

**Headers:**

| Поле | Описание |
|------|----------|
| `Content-Type` | application/json |
| `deviceuuid` | ID устройства |

**Параметры/Body:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| phone | string | Номер телефона в формате +7XXXXXXXXXX | Из поля ввода |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Показ лоадера на кнопке, её блокировка |
| Успех (200) | Переход на экран ввода кода, старт таймера повторной отправки |
| Ошибка 400 | Снек с текстом "Неверный формат номера телефона" |
| Ошибка 429 | Снек с текстом из `message` (слишком много попыток) |
| Ошибка сети | Снек "Нет соединения. Проверьте подключение" |

---

### POST /auth/verify

**Триггер:** Тап на кнопку "Войти"

**Headers:**

| Поле | Описание |
|------|----------|
| `Content-Type` | application/json |
| `deviceuuid` | ID устройства |

**Параметры/Body:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| phone | string | Номер телефона | Из первого экрана |
| code | string | SMS-код | Из поля ввода |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Показ лоадера, блокировка кнопки |
| Успех (200) | Сохранение токенов, переход на главный экран |
| Ошибка 400 | Снек с текстом "Неверный код или истёк срок действия" |
| Ошибка 404 | Снек с текстом "Пользователь не найден" |
| Ошибка сети | Снек "Нет соединения. Проверьте подключение" |

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** Пользователь не авторизован, **Когда** Он вводит валидный номер телефона и запрашивает код, **Тогда** Приходит SMS с кодом |
| AC-002 | **Дано** Пользователь ввёл код из SMS, **Когда** Он нажимает "Войти", **Тогда** Он переходит на главный экран и видит свои бронирования |
| AC-003 | **Дано** Пользователь уже входил в приложение, **Когда** Он открывает приложение снова, **Тогда** Он автоматически входит без ввода кода |
| AC-004 | **Дано** Пользователь ввёл неверный код, **Когда** Он нажимает "Войти", **Тогда** Показывается сообщение об ошибке |
| AC-005 | **Дано** Пользователь запрашивает код слишком часто, **Когда** Он нажимает "Получить код", **Тогда** Показывается сообщение о превышении лимита |

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| Неверный формат телефона | Ввод номера | Снек с текстом ошибки от API |
| Неверный код | Ввод кода | Снек, очистка поля кода |
| Истёк код | Ввод кода | Снек "Код истёк", возврат на ввод телефона |
| Превышен лимит запросов | Запрос кода | Снек с таймером ожидания |
| Нет сети | Любой запрос | Снек с предложением проверить интернет |
| Серверная ошибка | Любой запрос | Снек "Попробуйте позже" |
