## Обзор

Подробная информация о бронировании: дата, время, зона, инструктор, стоимость, статус, возможность отмены.

### User Story

> Как **клиент**, я хочу **просмотреть детали бронирования**, чтобы **знать информацию о тренировке**.

### Бизнес-ценность

- Прозрачность информации о бронированиях
- Удобная отмена бронирований
- Снижение количества пропущенных тренировок

## Используемые запросы

> Все API-запросы экрана с полным описанием параметров и обработки ответов.

### GET /bookings/{bookingId}

**Тип:** REST  
**Метод:** GET  
**Спецификация:** [openapi.yaml](../api/openapi.yaml) → `getBookingById`

**Триггер:** Инициализация / Pull-to-refresh

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `bookingId` | path (string) | Да | URL | ID бронирования |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Скелетон |
| Успех | 200 OK | Отобразить детали бронирования |
| HTTP 401 | — | Redirect на SCR-01 |
| HTTP 404 | Бронирование не найдено | Error state "Бронирование не найдено" |
| Сеть | Нет соединения | Error state с кнопкой "Обновить" |

**Метод:** PATCH  
**Спецификация:** [openapi.yaml](../api/openapi.yaml) → `cancelBooking`

**Триггер:** Подтверждение отмены в Bottom Sheet

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `bookingId` | path (string) | Да | URL | ID бронирования |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Лоадер на кнопке |
| Успех | 200 OK | Обновить статус, показать success |
| HTTP 400 | Невозможно отменить | Снек "Невозможно отменить" |
| HTTP 401 | — | Redirect на SCR-01 |
| HTTP 404 | Бронирование не найдено | Error state |
| Сеть | Нет соединения | Снек "Нет соединения" |

> - **Условия доступности:** Для кнопок и интерактивных элементов указать условия активности/видимости после таблицы.

### 1. Header

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Кнопка "Назад" | Стрелка влево | — | — | Возврат на SCR-05 |
| Заголовок | "Детали бронирования" | — | — | — |

### 2. Блок даты и времени

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Дата | Заголовок | `slot.startTime` → форматирование | — | — |
| Время | Текст | `slot.startTime` - `slot.endTime` | — | — |

**Логика:**
- Формат даты: "15 янв, понедельник"
- Формат времени: "10:00 - 12:00"

### 3. Блок зоны

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Зона | Текст | `slot.zone.name` | — | — |
| Уровень | Бейдж | `slot.zone.level` | — | Easy/Medium/Hard |
| Продолжительность | Текст | `slot.zone.durationMinutes` | — | X часов |

**Логика:**
- Уровень отображается цветом: Easy=#4CAF50, Medium=#FF9800, Hard=#F44336

### 4. Блок информации о слоте

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Тренер | Текст | `slot.instructor.name` | — | — |
| Снаряжение | Текст | `equipmentType` | — | own/rental |
| Арендованное оборудование | Текст | `equipment.name` | — | (если rental) |

**Логика:**
- Если `equipmentType = own`: "Своё"
- Если `equipmentType = rental`: "Аренда: {equipment.name}"

### 5. Блок бронирования

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| ID | Текст | `id` | — | — |
| Гости | Текст | — | — | "+ X гостей" (если есть) |
| Статус | Бейдж | `status` | — | Цвет: success/error/warning |
| Цена | Заголовок | `price` | — | Формат: "X ₽" |
| Штраф | Текст | `cancellationPenalty` | — | (только при отмене) |

**Логика:**
- Статус отображается цветом и текстом:
  - `confirmed` = зелёный "Подтверждено"
  - `cancelled_by_client_early` = красный "Отменено (бесплатно)"
  - `cancelled_by_client_late` = красный "Отменено (со штрафом)"
  - `cancelled_by_gym` = серый "Отменено скалодромом"

### 6. Footer

| Элемент | Описание | Источник данных | Валидация | Действие |
|---------|----------|-----------------|-----------|----------|
| Кнопка "Отменить" | Danger Button | — | — | Открыть BS-02 |
| Кнопка "На главную" | Secondary Button | — | — | Возврат на SCR-02 |

**Логика:**
- Кнопка "Отменить" видна только для `status = confirmed`
- При отмене: проверка времени до начала слота
  - ≥2ч до начала: бесплатно
  - <2ч до начала: штраф 10%

**Условия доступности:**
- Кнопка "Отменить" активна, если: `status = confirmed`

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** авторизованный пользователь, **Когда** открытие экрана, **Тогда** загружаются детали бронирования | P0 |
| AC-002 | **Дано** бронирование загружено, **Тогда** отображаются дата, время, зона, тренер, цена | P0 |
| AC-003 | **Дано** статус confirmed, **Тогда** кнопка "Отменить" активна | P0 |
| AC-004 | **Дано** подтверждение отмены, **Тогда** бронирование отменяется | P0 |
| AC-005 | **Дано** успешная отмена, **Тогда** обновляется статус | P0 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** ошибка сети, **Когда** открытие экрана, **Тогда** error state | P0 |
| AC-N02 | **Дано** 404, **Тогда** показывается "Бронирование не найдено" | P0 |
| AC-N03 | **Дано** статус не confirmed, **Тогда** кнопка "Отменить" скрыта | P0 |
| AC-N04 | **Дано** 401, **Тогда** редирект на авторизацию | P0 |

### Граничные условия (Edge Cases)

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** отмена >2ч до начала, **Тогда** штраф = 0 | P1 |
| AC-E02 | **Дано** отмена <2ч до начала, **Тогда** штраф = 10% | P1 |
| AC-E03 | **Дано** отмена скалодромом, **Тогда** причина отмены | P1 |
