## Обзор

Экран авторизации позволяет пользователю войти в приложение по номеру телефона через SMS-код. Клиент вводит номер телефона, получает SMS-код и входит в приложение.

### User Story

> Как **новый/существующий клиент**, я хочу **войти по номеру телефона**, чтобы **получить доступ к функционалу приложения**.

### Бизнес-ценность

- Регистрация новых клиентов
- Быстрый вход без пароля
- Снижение барьера входа

## Используемые запросы

> Все API-запросы экрана с полным описанием параметров и обработки ответов.

### /auth/request-code

**Тип:** REST  
**Метод:** POST  
**Спецификация:** [authentication.yaml](../api/openapi.yaml) → `requestVerificationCode`

**Триггер:** Тап на кнопку "Получить код"

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `phone` | string | Да | Ввод пользователя | Номер телефона в формате +7XXXXXXXXXX |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Лоадер на кнопке, кнопка заблокирована |
| Успех | `success = true` | Переход к вводу кода / Автоматическая отправка на verify |
| HTTP 400 | Неверный формат | Снек "Неверный формат номера телефона" |
| HTTP 429 | Превышен лимит | Снек "Слишком много попыток. Попробуйте позже" |
| Сеть | Нет соединения | Снек "Нет соединения. Проверьте подключение" |

---

### /auth/verify

**Тип:** REST  
**Метод:** POST  
**Спецификация:** [authentication.yaml](../api/openapi.yaml) → `verifyCode`

**Триггер:** Ввод SMS-кода (ручной или автоматический)

**Параметры:**

| Параметр | Тип | Обязательность | Источник | Описание |
|----------|-----|----------------|----------|----------|
| `phone` | string | Да | Ввод пользователя | Номер телефона |
| `code` | string | Да | Ввод пользователя | SMS-код из 4-6 цифр |

**Обработка ответа:**

| Результат | Условие | UI-реакция |
|-----------|---------|------------|
| Загрузка | — | Лоадер на кнопке |
| Успех | 200 OK + `accessToken` | Сохранить токены, переход на SCR-02 |
| HTTP 400 | Неверный код | Снек "Неверный код. Попробуйте ещё раз" |
| HTTP 404 | Пользователь не найден | Снек "Пользователь не найден" |
| Сеть | Нет соединения | Снек "Нет соединения. Проверьте подключение" |

## Критерии приёмки

### Позитивные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-001 | **Дано** валидный номер телефона, **Когда** пользователь нажимает "Получить код", **Тогда** отправляется запрос `/auth/request-code` | P0 |
| AC-002 | **Дано** успешный ответ `/auth/verify`, **Тогда** пользователь перенаправляется на главный экран | P0 |

### Негативные сценарии

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-N01 | **Дано** невалидный номер телефона, **Когда** пользователь нажимает "Получить код", **Тогда** отображается ошибка валидации | P0 |
| AC-N02 | **Дано** ошибка сети, **Когда** запрос к API, **Тогда** отображается error state | P0 |
| AC-N03 | **Дано** превышен лимит запросов (429), **Когда** запрос кода, **Тогда** отображается сообщение "Слишком много попыток" | P1 |

### Граничные условия (Edge Cases)

| ID | Критерий | Приоритет |
|----|----------|-----------|
| AC-E01 | **Дано** уже авторизованный пользователь, **Когда** открытие экрана, **Тогда** автоматический переход на главный экран | P1 |
| AC-E02 | **Дано** пользователь уже вводил номер, **Когда** открытие экрана, **Тогда** поле предзаполнено | P2 |
